## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead:

- Contact the maintainer privately via GitHub
- Or open a GitHub Security Advisory if available

Please include:

- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Any relevant logs or screenshots

---

## Responsible Disclosure

We ask that you:

- Allow reasonable time for investigation and remediation
- Do not publicly disclose vulnerabilities before a fix is available

We aim to respond to security reports within a reasonable timeframe,
depending on severity and complexity.

---

## Scope of Security Issues

This policy applies to:

- Configuration handling
- Tariff calculations
- Input validation
- Data exposure within Home Assistant

This policy does **not** cover:

- Issues caused by misconfiguration of Home Assistant itself
- Third-party integrations or custom components
- Insecure network setups
